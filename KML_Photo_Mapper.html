⬡ USER GUIDE

Overview

Upload a KML file of site pins and field photos exported from Fulcrum. The tool extracts GPS from EXIF, groups photos by UUID, matches each group to the nearest KML pin, and exports a structured ZIP split into DAP / FDH / HH / TB / MISC folders.

Blue triangle pins — KML site locations from your .kml file (black outline)
Green circle pins — Assigned photo groups (dark green outline)
Red circle pins — Unassigned photo groups (black outline)
Step-by-step workflow
1
Upload your KML file — Drop or click the KML zone. Blue triangle pins appear on the map and it auto-fits to show them all. The KML panel collapses once both files are loaded.
2
Upload your field photos — Photos must follow the naming format {record_id} - {label}.jpg and must contain GPS in EXIF. A manual binary GPS parser runs as fallback if exif-js misses GPS tags. Photos without GPS are skipped.

Fulcrum setup: Set Media Fields to File Name and Media File Name to {record_id} - {label} before exporting.
3
Auto-assignment runs immediately — Each group is matched to the nearest KML pin within 15 metres. TB groups with a fully-named KML pin (e.g. TB 295 Spring Hill Dr) are assigned instantly with no geocoding needed. Unassigned groups appear at the top of the sidebar list in red.
4
Review & assign — Use the ⟳ Auto-assign All bulk button for remaining unassigned groups, or click any group pin / assign button to open the floating assignment modal. The ⬡ Unassigned filter and search bar help you find groups quickly.
5
Export — Once at least one group is assigned, the Download button appears. Enter a site name and get a ZIP with organised subfolders. Same-name groups are merged automatically.
↺
Restart — The ↺ Restart — Clear All button appears as soon as either file is loaded. Click once to arm it (turns red), click again within 3 seconds to clear everything and start fresh.
Photo filtering rules

The following photos are always discarded before processing:

BORE  ·  CONCRETE  ·  ASPHALT  ·  PAVERS  ·  SOD
INCIDENT  ·  WATER  ·  FAILED

Conditional rules:

Photos label → discarded unless filename also contains DAP, FDH, HANDHOLE, or TOBY BOX
White Line / Power Meter → routed to FDH pins and FDH folder only

Automatic assignment (15m limit)

Each GPS-equipped group is matched to the nearest eligible KML pin within 15 metres. Photo-type routing applies:

DAP photos → DAP pins only
FDH / Handhole photos → FDH / HH pins respectively
White Line / Power Meter → FDH pins only
Toby Box photos → TB / TOBY pins only
All others → any pin (MISC bucket)
Smart TB naming: If a KML pin is already named TB 295 Spring Hill Dr, any Toby Box group within 15m inherits that full name automatically — no geocoding or manual entry needed.
Order doesn't matter. Upload photos first or KML first — auto-assignment runs either way once both are loaded.
Floating assignment modal
1
Click a map pin or the ⊕ Assign pin name / ✎ Change button inside an expanded group to open the floating modal. It appears attached to the left edge of the sidebar.
2
Pins within 15m appear at full opacity at the top. Pins beyond 15m appear below an orange ⚠ Beyond 15m separator for manual override only.
3
Type a custom name and press Enter or ✓ Use, or click any pin from the list. All names are automatically uppercased.
4
Click ✕ Cancel or click outside the modal to dismiss without changes.
TB address: Toby Box groups without a pre-named KML pin show a ⚠ needs address badge and an address input field with a Google Maps link for reference.
Downloading the ZIP
1
The ⬇ Download Renamed ZIP button appears once at least one group has an assigned name.
2
Enter a site name when prompted — this becomes the ZIP filename (e.g. D-HBR204.zip).
3
The ZIP is built entirely in your browser. Groups with the same pin name are merged into one folder automatically. The bucket (DAP/FDH/HH/TB/MISC) is determined by the pin name first, then by photo content.
ZIP folder structure
D-HBR204.zip
  ├── DAP/
  │   └── DAP-D-01/ · DAP, DAP Photos
  ├── FDH/
  │   └── FDH-HH01/ · FDH, White Line, Power Meter
  ├── HH/
  │   └── HH-01/ · Handhole photos
  ├── TB/
  │   └── TB 295 SPRING HILL DR/ · Toby Box photos
  └── MISC/
      └── SITE-NAME/ · anything else
Tips & notes
Duplicate filenames within the same folder are automatically renamed with a numeric suffix (e.g. photo_2.jpg).
Reloading KML prompts whether to also clear existing photos and groups.
Re-uploading photos with the same filename overwrites the existing copy — useful for correcting individual photos without re-uploading everything.
GPS fallback parser: If exif-js fails to read GPS tags (common with some Android/Fulcrum captures), a manual JPEG binary parser scans the GPS IFD directly as a fallback.
Nothing leaves your device. All processing — EXIF reading, GPS extraction, ZIP building — happens entirely in your browser.
